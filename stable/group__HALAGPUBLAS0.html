Wrapper to GPU-BLAS matrix addition Xgeam().

The GPU-BLAS library includes the BLAS-like extension Xgeam() that computes \( C = \alpha op(A) + \beta op(B) \).

  • transa and transb are lower-upper case N/T/C indicating the op applied to A and B
  • op can be identity (N), transpose (T), or conjigate-transpose (C)
  • A and B are an M by N (or N by M) matrices with leading dimensions lda and ldb
  • lda and ldb are minimum M for transa = N and N otherwise
  • C is ldc by N with minimum ldc of M

Overloads:

geam(engine, transa, transb, M, N, alpha, A, beta, B, C, lda=-1, ldb=-1, ldc=-1);
geam(engine, transa, transb, M, N, alpha, A, lda, beta, B, ldb, C, ldc);

The method is GPU-BLAS extension hence only the hala::gpu_engine and hala::mixed_engine overloads will perform any action, the hala::cpu_engine overload will be a no-op.

Wrapper to GPU-BLAS matrix addition Xdgmm().

The GPU-BLAS library includes the BLAS-like extension Xdgmm() that computes \( C = A D \) or \( C = D A \), i.e., a dense matrix multiplied by a diagonal matrix.

  • side is upper/lower case L/R indicating whether D is on the left (L) or right (R) of A
  • A is an M by N matrix with leading dimension lda (minimum M)
  • x holds the diagonal entries of D in a stride of incx
  • the active entries in x must be either M (for side left) or N (for side right)
  • C is the resulting matrix with leading dimension ldc (minimum M)

Overloads:

dgmm(engine, side, M, N, A, x, C, lda=-1, incx=-1, ldc=-1);
dgmm(engine, side, M, N, A, lda, x, incx, C, ldc);

The method is GPU-BLAS extension hence only the hala::gpu_engine and hala::mixed_engine overloads will perform any action, the hala::cpu_engine overload will be a no-op.

Wrapper to GPU-BLAS matrix addition Xtpttr(), converts triangular-packed to triangular-regular formats.

Takes a triangular packed matrix and converts it into a regular N by N triangular matrix.

  • uplo is lower/upper case L/U indicating whether to use the lower (L) or upper (U) part of A
  • N is the number of rows and columns of the matrix
  • AP has N (N+1) / 2 entries holding the packed matrix
  • A is a matrix lda by N and the upper or lower part of the matrix will be rewritten
  • lda is minimum N, and A is always resized (if the size is not sufficient already) See also the inverse operation hala::tr2tp().

Overloads: the method can be used with a hala::mixed_engine too.

Wrapper to GPU-BLAS matrix addition Xttttp(), converts triangular-regular to triangular-packed formats.

Takes a triangular matrix and converts it into packed format.

  • uplo is lower/upper case L/U indicating whether to use the lower (L) or upper (U) part of A
  • N is the number of rows and columns of the matrix
  • A is a matrix lda by N, minimum lda is N
  • AP will be resized and overwritten with the N (N+1) / 2 entries holding the packed matrix See also the inverse operation hala::tp2tr().

Overloads:

tr2tp(engine, uplo, N, A, lda, AP);
tr2tp(engine, uplo, N, A, AP, lda = -1);

The lda will assume the minimum value.
