Detailed Description

Methods that combine multiple BLAS, cuBLAS and/or manually implemented methods the perform some simple operations, such as batch dot-product or matrix transpose.

Compute the scaled vector sum of a batch of one dimensional vectors and scalars, where the vectors are stored contiguously.

Computes \( y_i = \alpha_c \alpha_i x_i + y_i \) for a set of vectors.

  • x and y have the same size and store a batch of vectors contiguously, i.e., the first vector is in the first strip of length x.size() / batch_size
  • alpha is a vector of length batch_size indicating the scalars for each vector pair, i.e., alpha_i
  • alphac is a constant that multiplies all scalars

Overloads

batch_axpy(engine, batch_size, alphac, alpha, x, y);

Computes the dot-product for a batch of one dimensional vectors.

Computes \( result_i = dot( x_i, y_i ) \) in either real or conjugate modes.

  • x and y have the same size and store a batch of vectors contiguously, i.e., the first vector is in the first strip of length x.size() / batch_size
  • result will be resized to at least batch_size and will hold the dot-products

Overloads

batch_dot(engine, batch_size, alphac, alpha, x, y);

Scales a batch of vectors by a set of constants.

Assuming that x has a batch of vectors stored contiguously and that alpha holds a set of scalars, this scales each vector by the corresponding scalar. It is equivalent to calling hala::scal() on each individual vector, but the CUDA version of the operation is performed with hala::dgmm().

Overloads:

batch_scal(batch_size, alpha, x);
batch_scal(engine, batch_size, alpha, x);

Returns the norm of the largest vector in a batch.

Here x represents a batch of vectors stored contiguously, this method returns the norm of the largest vector.

  • batch_size is a non-negative number indicating the size of the batch
  • x is the batch of vectors, there is no check if the size is an even multiple of the batch size, but it is assumed that the size of each vector is size-of-x divided by batch_size

Overloads:

batch_max_norm2(batch_size, x);
batch_max_norm2(engine, batch_size, x);

Implementations are provided for all compute engines, when called without an engine the cpu_engine will be assumed.
