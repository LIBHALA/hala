Detailed Description

BLAS Level 3
Level 3 operations are defines as those that include a two or more matrices. Examples of such operations are matrix-matrix product and triangular solve with multiple right hand sides.

Wrapper to BLAS triangular matrix-matrix multiply xtrmm().

Computes \( B = \alpha op(A) B \) or \( B = \alpha B op(A) \), where op is either identity or, conjugate or real transpose operators. Here A is assumed to be upper or lower triangular matrix with either unit or non-unit diagonal.

  • side must be lower or upper case L or R, indicates whether A is on the left (L) or right (R) of B
  • uplo must lower or upper case U or L, indicates whether to work with the upper (U) or lower (L) part of A
  • trans must be lower or upper case N (identity), T (transpose), or C (conjugate transpose), indicates op
  • diag must be lower or upper case U or N, indicates whether A has unit (U) or non-unit (N) diagonal
  • M indicates the number of rows of B
  • N indicates the number of columns of B
  • only the array elements of A indicated by uplo and diag are accessed, the rest is not read
  • lda and ldb are the leading dimensions of A and B
  • lda is minimum M for side L or N for size R, ldb is minimum M

Overloads:

hala::trmm(side, uplo, trans, diag, M, N, alpha, A, B, lda = -1, ldb = -1);
hala::trmm(engine, side, uplo, trans, diag, M, N, alpha, A, lda, B, ldb);
hala::trmm(engine, side, uplo, trans, diag, M, N, alpha, A, B, lda = -1, ldb = -1);

The leading dimensions assume the minimum acceptable value, i.e., as if the matrices have no padding.

Wrapper to BLAS symmetric or Hermitian rank-k matrix update xsyrk()/xherk().

Computes \( C = \alpha op(A) A + \beta C \) or \( C = \alpha A op(A) + \beta C \). Here C is assumed to be a symmetric or Hermitian matrix and only the upper or lower part of C is referenced.

  • conjugate indicates whether C is conjugate-symmetric (Hermitian) or non-conjugate symmetric
  • uplo is lower/upper case U/L, indicates whether to use the upper (U) or lower (L) part of C
  • trans is lower/upper case N/T/C, indicates whether the op is on the first (T/C) or second (N) occurrence of A
  • trans cannot be T if C is a Hermitian matrix
  • N is the size of the square matrix C
  • K indicates the number of rows (trans is T) or columns (trans is N) of A
  • lda, ldc are the leading dimensions
  • lda is minimum N if trans is N and minimum K if trans is T, ldc is minimum N

Overloads:

hala::syrk(uplo, trans, N, K, alpha, A, beta, C, lda = -1, ldc = -1);
hala::herk(uplo, trans, N, K, alpha, A, beta, C, lda = -1, ldc = -1);
hala::syrk(engine, uplo, trans, N, K, alpha, A, lda, beta, C, ldc);
hala::herk(engine, uplo, trans, N, K, alpha, A, lda, beta, C, ldc);
hala::syrk(engine, uplo, trans, N, K, alpha, A, beta, C, lda = -1, ldc = -1);
hala::herk(engine, uplo, trans, N, K, alpha, A, beta, C, lda = -1, ldc = -1);

The leading dimensions assume the minimum acceptable value, i.e., as if the matrices have no padding. The syrk() and herk() variants differ in whether the output is symmetric or conjugate symmetric (Hermitian).

Wrapper to BLAS symmetric or Hermitian 2x rank-k matrix update xsyr2k()/xher2k().

Computes \( C = \alpha op(A) B + \bar \alpha op(B) A + \beta C \) or \( C = \alpha A op(B) + \bar \alpha B op(A) + \beta C \). Here C is assumed to be a symmetric or Hermitian matrix and only the upper or lower part of C is referenced.

  • conjugate indicates whether C is conjugate-symmetric (Hermitian) or non-conjugate symmetric
  • uplo is lower/upper case U/L, indicates whether to use the upper (U) or lower (L) part of C
  • trans is lower/upper case N/T/C, indicates whether the op is on the first (T/C) or second (N) occurrence of A
  • N is the size of the square matrix C, ldc is minimum N
  • K indicates the number of rows (trans is T) or columns (trans is N) of A and B
  • lda and ldb are the leading dimensions, minimum the number of rows indicated above

Overloads:

hala::syr2k(uplo, trans, N, K, alpha, A, B, beta, C, lda=-1, ldb=-1, ldc=-1);
hala::her2k(uplo, trans, N, K, alpha, A, B, beta, C, lda=-1, ldb=-1, ldc=-1);
hala::syr2k(engine, uplo, trans, N, K, alpha, A, lda, B, ldb, beta, C, ldc);
hala::her2k(engine, uplo, trans, N, K, alpha, A, lda, B, ldb, beta, C, ldc);
hala::syr2k(engine, uplo, trans, N, K, alpha, A, B, beta, C, lda=-1, ldb=-1, ldc=-1);
hala::her2k(engine, uplo, trans, N, K, alpha, A, B, beta, C, lda=-1, ldb=-1, ldc=-1);

The leading dimensions assume the minimum acceptable value, i.e., as if the matrices have no padding. The difference between syr2k() and her2k() variants lies in the complex case where the matrix C is either symmetric or conjugate-symmetric (Hermitian).

Wrapper to BLAS symmetric or Hermitian matrix multiply xsymm()/xhemm().

Computes \( C = \alpha A B + \beta C \) or \( C = \alpha B A + \beta C \). Here A is assumed to be a symmetric matrix and only the upper or lower part of A is referenced.

  • conjugate indicates whether A is conjugate-symmetric (Hermitian) or non-conjugate symmetric
  • side is lower/upper case L/R, indicates whether A is on the left (L) or right (R) of B
  • uplo is lower/upper case U/L, indicates whether to use the upper (U) or lower (L) part of C
  • M is the number of rows of B and C
  • N is the number of columns of B and C
  • lda, ldb and ldc are the leading dimensions
  • lda is minimum M when side is L and N when side is R, ldb and ldc are minimum M

Overloads:

hala::symm(side, uplo, M, N, alpha, A, B, beta, C, lda=-1, ldb=-1, ldc=-1);

hala::symm(engine, side, uplo, M, N, alpha, A, B, beta, C, lda=-1, ldb=-1, ldc=-1);
hala::hemm(engine, side, uplo, M, N, alpha, A, B, beta, C, lda=-1, ldb=-1, ldc=-1);

The leading dimensions assume the minimum acceptable value, i.e., as if the matrices have no padding. The difference between symm() and hemm() variants lies in the complex case where the matrix is either symmetric or conjugate-symmetric (Hermitian).

Wrapper to BLAS general matrix multiply xgemm().

Computes \( C = \alpha op(A) op(B) + \beta C \), for general matrices, where op can be identity, conjugate or transpose operations (there are two operations applied independently).

  • transa and transb must be lower/upper case N (identity), T (transpose), or C (conjugate transpose)
  • transa and transb indicate op on A and B respectively
  • M is the rows of C and op(A)
  • N is the columns of C and op(B)
  • K indicates the inner dimension of the product
  • lda, ldb or ldc are the leading dimensions
  • lda is minimum M when transa is N and minimum K when transa is T or C
  • ldc is minimum K when transb is N and minimum N when transb is T or C
  • ldc is minimum M

Overloads:

hala::gemm(transa, trnasb, M, N, K, alpha, A, B, beta, C, lda=-1, ldb =-1, ldc=-1);
hala::gemm(engine, transa, trnasb, M, N, K, alpha, A, lda, B, ldb, beta, C, ldc);
hala::gemm(engine, transa, trnasb, M, N, K, alpha, A, B, beta, C, lda=-1, ldb =-1, ldc=-1);

The leading dimensions assume the minimum acceptable value, i.e., as if the matrices have no padding.
