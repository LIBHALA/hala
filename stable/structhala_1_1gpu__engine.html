Detailed Description

Core engine class, deals with handles and data transfer.

The engine class is a wrapper around the cuBlas and cuSparse handles and holds basic meta-data such as the associated GPU device ID. The engine-type is used in template overloads to indicate the use of GPU libraries and the engine provides basis I/O methods for sending/receiving vectors, which is useful in making sure the vectors are associated with the correct GPU device.
