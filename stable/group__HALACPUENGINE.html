VectorLike const &

Returns a default vector with the default constructor.

Uses both hala::define_vdefault and hala::vector_constructor to make a default (probably) empty vector. The call will instantiate to return an vector for the appropriate engine and the value type will be copied from the provided vector-like. Note that the data is not copied, the input vector is used to infer only the type.

Overloads:

auto x = new_vector(engine, t);
auto y = new_vector(t); // t is ordinary vector-like class
auto z = new_vector(t); // t is hala::engined_vector

Overloads are provided for all engine types, for hala::engined_vectors and for vector only inputs. In the examples above, x will have the default vector type for the engine, see hala::define_vdefault. The y vector will have the default vector type associated with the hala::cpu_engine. The z will have the default type associated with the engine of t and z will alias the engine of t.

Fills the first num_entries of a vector with zeros.

Used to set the initial iterate of solver algorithms.

Overloads:

set_zero(num_entries, x);
set_zero(engine, num_entries, x);

The overloads include all compute engines and the engined_vector class.

Create an array wrapper for the corresponding engine.

Creates a wrapper around an array so that the wrapper "knows" the array size. This makes it safer to call HALA methods directly with raw-arrays.

Overloads:

wrap_array(arr, num_entries);
wrap_array(engine, arr, num_entries);

Variants of this method are provided for each engine class resulting in appropriate corresponding wrappers. If called without an engine, the method assumes the cpu_engine.
