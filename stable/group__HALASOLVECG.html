Detailed Description

A series of templates that implement the preconditioned conjugate-gradient method with several options to choose from different preconditioners.

General preconditioned conjugate-gradient solver.

Solves \( A x = b \) where A is a sparse symmetric-positive-definite matrix and P is an appropriate preconditioner. The iteration is terminated when either the residual drops below the tolerance or the maximum number of iterations is reached.

  • pntr, indx, vals describe the sparse matrix A in row compressed format
  • b specifies the right-hand-side of the equation
  • x is both the initial guess for the iteration and the final computed solution, if the size of x is incorrect it will be resized and reset to zero
  • precon(p, r) takes two vectors of the internal type and overwrites r with the \( P^{-1} p \), see hala::preconditioner

Overloads:

solve_cg(stop, pntr, indx, vals, precon, b, x);
solve_cg(engine, stop, pntr, indx, vals, precon, b, x);

Conjugate-gradient solver with ILU preconditioner, single right-hand-side.

Solves \( A x = b \) using the conjugate-gradient method.

  • stop defines the tolerance (measured as the norm of the residual) and the maximum number of iterations
  • pntr, indx, and vals describe a symmetric positive definite sparse matrix in row-compressed format
  • ilu is precomputed ILU factor matching the engine, e.g., cpu_ilu, cuda_ilu or mixed_ilu
  • b is the right-hand-side of the equation
  • x is the initial guess to start the iteration, if the size of x is insufficient it will be resized and scaled by zero
  • returns the total number of iterations

Overloads:

solve_cg_ilu(stop, pntr, indx, vals, b, x);
solve_cg_ilu(stop, pntr, indx, vals, ilu, b, x);
solve_cg_ilu(engine, stop, pntr, indx, vals, b, x);
solve_cg_ilu(engine, stop, pntr, indx, vals, ilu, b, x);

The methods work with all compute engines, if the ilu input is missing the overload will perform the ilu factorization "on-the-fly" and discard it after the code has completed.

Note: that the engine for the ILU factor and the rest of the vectors must match.
