Query the size of the workspace needed for sparse gemv operation.

Some implementations of sparse operations require additional workspace memory which can be pre-allocated by the user to avoid repeated allocate/deallocate operations that can be expensive, e.g., in an iterative solver context. The user provided workspace needs to have size at least equal to the size returned here, the size is measured in entries of the matrix type, e.g., float or std::complex<double>.

See hala::sparse_gemv().

Sparse matrix vector multiplication.

Computes \( y = \alpha A x + \beta y \).

  • The matrix is stored in row-compressed format and represented by the variables pntr, indx, vals
  • x is a vector of the same type as vals
  • y is a vector of the same type as x and with size matching the number of rows
  • alpha and beta are scalars convertible to the type of the vectors

Note: the number of columns of the matrix is implicit and the size of x cannot be checked.

Overloads:

sparse_gemv(trans, M, N, alpha, pntr, indx, vals, x, beta, y);
sparse_gemv(engine, trans, M, N, alpha, pntr, indx, vals, x, beta, y);
sparse_gemv(sparse_matrix, trans, alpha, x, beta, y);
sparse_gemv(sparse_matrix, trans, alpha, x, beta, y, workspace);
cpu_sparse_matrix::gemv(trans, alpha, x, beta, y);
cpu_sparse_matrix::gemv(trans, alpha, x, beta, y, workspace);

The sparse_matrix can be either of the classes hala::cpu_sparse_matrix or hala::gpu_sparse_matrix, the last overload is a member of method.

Sparse matrix-matrix multiplication.

Computes \( C = \alpha op(A) op(B) + \beta C \) where A is a sparse matrix.

  • transa and transb are upper/lower case N/T/C indicating the op and A and B
  • the possible op variants are identity (N), transpose (T), and conjugate-transpose (C)
  • A is M by K for transa begin identity and K by M otherwise
  • B is K by N for transb begin identity and N by K otherwise
  • C is M by N
  • K is always the middle dimension of the product
  • ldc is minimum K when transb is N and minimum N when transb is T or C
  • ldc is minimum M
  • everything other than A is identical to hala::gemm()

Overloads:

sparse_gemm(transa, trnasb, M, N, K,
alpha, pntr, indx, vals, B, beta, C, ldb=-1, ldc=-1);
sparse_gemm(engine, transa, trnasb, M, N, K,
alpha, pntr, indx, vals, B, ldb, beta, C, ldc);
sparse_gemm(matrix, transa, trnasb, b_rows, b_cols,
alpha, B, ldb, beta, C, ldc);
sparse_gemm(matrix, transa, trnasb, b_rows, b_cols,
alpha, B, ldb, beta, C, ldc, workspace);
cpu_sparse_matrix::gemm(transa, trnasb, b_rows, b_cols,
alpha, B, ldb, beta, C, ldc);
cpu_sparse_matrix::gemm(transa, trnasb, b_rows, b_cols,
alpha, B, ldb, beta, C, ldc, workspace);

The leading dimensions assume the minimum acceptable value, i.e., as if the matrices have no padding. The last two overloads are member methods of hala::cpu_sparse_matrix and gpu_sparse_matrix and the matrix signatures deviate a bit from the (M, N, K) triple when defining the dimensions. The workspace is a containers (or raw-arrays) with size at least equal to hala::sparse_gemm_buffer_size() with user allocate memory; used primarily on the GPU.

Solves the triangular system of equations.

Computes \( A x = \alpha b \), where A is a sparse triangular matrix, and x and b are dense vectors.

  • trans is lower/upper case U/T/C indicating whether to take the transpose
  • tri is a triangular matrix constructed with hala::make_triangular_matrix()
  • b and x are the right-hand side and solution vectors
  • b must have size at least tri.rows()
  • x will be resized to tri.rows(), if the current size if not sufficient

Overloads are provided for triangular matrices constructed with all compute engines, note that x and b must be vectors located in CPU memory for the case when tri is constructed using the cpu_engine or the mixed_engine, and GPU memory for the cuda_engine.

Solves the triangular system of equations with multiple right hand sides.

Computes \( A X = \alpha B \), where A is a sparse triangular matrix, and X and B are general dense matrices.

  • transa and transb are upper/lower case N/T/C indicating whether to use identity (N), transpose or conjugate transpose on A and B
  • nrhs indicates the number of right-hand-sides to use
  • tri is a triangular matrix constructed with hala::make_triangular_matrix()
  • B (on entry) is the right-hand-side of the equation, and (on exit) it is overwritten with the solution
  • ldb is the leading dimension of B, minimum the number of rows of tri (when transb is N) or nrhs (transb is T or C)

Overloads are provided for triangular matrices constructed with all compute engines, note that x and b must be vectors located in CPU memory for the case when tri is constructed using the cpu_engine or the mixed_engine, and GPU memory for the cuda_engine.
